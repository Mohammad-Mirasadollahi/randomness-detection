# Randomness Detection

Detect how random a string looks on a **1–100 scale** (higher = more random). Combines bigram frequency, Shannon entropy, and DEFLATE compression via a logistic regression ensemble.

## One-line install

```bash
curl -fsSL https://raw.githubusercontent.com/Mohammad-Mirasadollahi/randomness-detection/main/install.sh | bash
```

The script clones the repo, installs dependencies, trains the model, and starts the API on **all network interfaces** (`0.0.0.0:8765`) with **open CORS** so any client can connect.

Already cloned?

```bash
./install.sh
```

**Restrict access** (edit `.env`, then `./install.sh --stop && ./install.sh`):

```bash
RANDOMNESS_HOST=127.0.0.1                         # localhost only
```

Full documentation: [Docs/README.md](Docs/README.md)

**Research model (paper-grade):** [LRD-Hybrid](Docs/research-lrd-hybrid.md) — LM + PMI + gradient boosting with ablation benchmark.

## Command line

`install.sh` puts a `randomness-detection` command on your `PATH` (works from any
directory, no venv activation):

```bash
randomness-detection "hello"                 # Score: 1   (natural)
randomness-detection "qzxwvbnmklpr" --json   # Score: 100 (random)
randomness-detection -f words.txt            # score a word list
randomness-detection-server --port 8765      # run the API server
```

See [Docs/cli.md](Docs/cli.md) for all options.

## Quick test (API)

After `./install.sh`, everything works immediately — no `source .env` needed:

```bash
randomness-detection "hello"
curl -s http://127.0.0.1:8765/health
curl -s -X POST http://127.0.0.1:8765/score \
  -H "Authorization: Bearer $RANDOMNESS_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"text":"hello"}'
```

Open a new terminal after install, or run once: `. ~/.config/randomness_detection/env.sh`

### Robustness — independent real-world stress test

```bash
PYTHONPATH=. .venv/bin/python test_robustness.py
```

Hand-curated data **not** generated by the training pipeline, guarding against band-aid fixes: real identifiers, brands, hyphenated phrases, digit-names vs UUIDs, SHAs, base64, API keys, DGA labels.

| Bucket | Metric | Production | LRD-Hybrid |
|--------|--------|------------|------------|
| core_natural | false-positive rate | **8%** | **5%** |
| clear_random | false-negative rate | **0%** | **0%** |

The model judges the **linguistic structure** of a string (dictionary coverage + character statistics), not its surface form. Casing, separators, and digits are decorrelated from the label in training, so identifiers like `getUserById` and phrases like `the-quick-brown-fox` are not mistaken for random. See [Docs/README.md#benchmarks](Docs/README.md#benchmarks) for the root-cause analysis and known limitations.

### Real-world data — DGA detection (production use case)

```bash
PYTHONPATH=. .venv/bin/python test_real_world_data.py
```

Real public datasets, unrelated to training: [Tranco top-1M](https://tranco-list.eu/) legit domains vs [andrewaeva/DGA](https://github.com/andrewaeva/DGA) 800K+ real malware domains.

| Measure (real data) | Result |
|---------------------|--------|
| Random-character DGA recall (real malware) | **95.4%** |
| Legit vs random-char DGA, clean ROC-AUC | **0.966** |
| Dictionary-DGA recall (families 4 & 7) | **42–65%** (word-count signal) |
