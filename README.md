# Randomness Detection

Detect how random a string looks on a **1–100 scale** (higher = more random). Combines **bigram frequency**, **Shannon entropy**, **DEFLATE compression**, **dictionary segmentation**, **character language-model perplexity**, and **word PMI** via a calibrated **gradient-boosting ensemble**.

## One-line install

```bash
curl -fsSL https://raw.githubusercontent.com/Mohammad-Mirasadollahi/randomness-detection/main/install.sh | bash
```

The script clones the repo, installs dependencies, trains the model, and starts the API on **all network interfaces** (`0.0.0.0:8765`) with **open CORS** so any client can connect.

Already cloned?

```bash
./install.sh
```

**Restrict access** (edit `.env`, then `./install.sh --stop && ./install.sh`):

```bash
RANDOMNESS_HOST=127.0.0.1                         # localhost only
```

Full documentation: [Docs/README.md](Docs/README.md)

## Command line

`install.sh` puts a `randomness-detection` command on your `PATH` (works from any
directory, no venv activation):

```bash
randomness-detection "hello"                 # Score: 1   (natural)
randomness-detection "qzxwvbnmklpr" --json   # Score: 100 (random)
randomness-detection -f words.txt            # score a word list
randomness-detection-server --port 8765      # run the API server
```

See [Docs/cli.md](Docs/cli.md) for all options.

## Quick test (API)

After `./install.sh`, everything works immediately — no `source .env` needed:

```bash
randomness-detection "hello"
curl -s http://127.0.0.1:8765/health
curl -s -X POST http://127.0.0.1:8765/score \
  -H "Authorization: Bearer $RANDOMNESS_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"text":"hello"}'
```

Open a new terminal after install, or run once: `. ~/.config/randomness_detection/env.sh`

### Robustness — independent real-world stress test

```bash
PYTHONPATH=. .venv/bin/python test_robustness.py
PYTHONPATH=. .venv/bin/python test_ensemble.py
```

Hand-curated data **not** generated by the training pipeline: real identifiers, brands, hyphenated phrases, digit-names vs UUIDs, SHAs, base64, API keys, DGA labels.

| Bucket | Metric | Result |
|--------|--------|--------|
| core_natural | false-positive rate | **5%** |
| clear_random | false-negative rate | **0%** |
| dictionary-DGA (4+ words, curated) | recall | **100%** |

Feature ablation: `PYTHONPATH=. .venv/bin/python ablation_benchmark.py`

The model judges **linguistic structure** (dictionary coverage, word count, LM perplexity, PMI), not surface form alone. See [Docs/README.md#benchmarks](Docs/README.md#benchmarks) for analysis and known limitations.
